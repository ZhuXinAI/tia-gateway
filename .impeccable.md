## Design Context

### Users
Developers and operators use `tia-gateway` to connect local agents to real communication channels and lightweight browser clients. They are usually setting this up in a terminal, validating that the gateway works, and then using the web UI as a quick control surface rather than a full product workspace.

### Brand Personality
Quietly technical, trustworthy, and low-friction. The interface should feel calm and competent instead of flashy, with an emphasis on clarity, quick verification, and operational confidence.

### Aesthetic Direction
Lean, text-first, and tool-oriented. Prefer a restrained light theme with warm neutrals, crisp structure, and subtle accents that make the interface feel intentional without competing with the conversation itself.

### Design Principles
1. Make first-run access obvious: users should understand how to unlock and start chatting in seconds.
2. Keep the chrome calm: the conversation and channel state should matter more than decorative UI.
3. Reward technical users with precision: clear labels, predictable controls, and transparent system hints.
4. Preserve mobility: the interface should adapt cleanly from laptop terminals to narrow mobile screens.
5. Treat trust as a design feature: auth, status, and session context should feel explicit but not noisy.
